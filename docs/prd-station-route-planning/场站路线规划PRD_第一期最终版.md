# 场站路线规划 PRD

## 司机端地图找充电场站｜第一期

| 项目 | 说明 |
| --- | --- |
| 产品名称 | 司机端地图 / 充电场站地图 |
| 适用端 | 司机小程序、PC 后台 |
| 小程序入口 | 底部任务栏「地图」入口 |
| PC 后台入口 | 基础信息 > 站点管理 |
| 第一期定位 | 地图找站、当前电价、距离展示、场站列表距离排序、外部导航、后台场站字段维护 |
| 字段原则 | 小程序仅展示后台「站点管理」已有字段与系统计算字段 |
| 版本 | V1.0 最终版 |

## 一、需求背景

新能源重卡司机在运输前、运输中、装卸货等待等场景下，需要快速查看可用充电场站，判断场站位置、电价、距离、充电桩数量等信息，并快速跳转外部地图导航。

当前后台已存在「基础信息 > 站点管理」能力，并已维护区域名称、区域编号、区域类型、类型、收/发货类型、所属部门、共享模式、半径、经度、纬度、位置、所属省份、所属城市、所属区、备注、开通状态、账号、修改人、修改时间、启用状态、结算主体、充电桩数量、电价、标准停留时长等字段。

第一期复用现有站点管理能力，将满足条件的正常充电场站数据同步给司机端地图使用。小程序端不新增后台没有维护来源的场站字段，避免前后端字段不一致。

## 二、需求分析

司机在找充电场站时，核心关注点主要包括：

1. 当前附近有哪些充电场站；

2. 场站距离当前位置多远；

3. 当前时段电价是多少；

4. 场站充电桩数量是否满足补能需求；

5. 场站地址是否准确；

6. 是否可以快速跳转外部地图导航。

因此，第一期功能重点为：

1. 地图展示正常充电场站；

2. 展示场站当前时段电价；

3. 展示用户当前位置到场站的距离；

4. 地图页支持定位 icon 回到用户当前位置并刷新地图数据；

5. 地图页支持「去充电」按钮，进入场站列表；

6. 场站列表按距离从近到远排序；

7. 支持场站列表搜索和筛选；

8. 支持外部地图导航；

9. PC 后台复用现有站点管理维护场站数据。

## 三、产品目标

1. 司机可在小程序地图页查看全部正常状态的充电场站。

2. 司机可基于用户手机定位查看附近充电场站。

3. 司机可在地图点位上查看当前时段电价和距离。

4. 司机可点击地图定位 icon，回到用户当前位置并刷新地图数据。

5. 司机可点击「去充电」按钮，加载并进入场站列表。

6. 场站列表页根据用户当前位置到场站的距离，从近到远排序。

7. 司机可通过场站列表页进行搜索、省份筛选、城市筛选、距离筛选。

8. 后台复用现有「基础信息 > 站点管理」，维护场站基础信息、分时电价、充电桩数量、标准停留时长、经纬度、地址等数据。

9. 小程序页面字段必须与后台站点管理已有字段保持一致。

## 六、功能说明

### 6.1 地图页

地图页为司机查找充电场站的主页面。

| 区域 | 内容 |
| --- | --- |
| 地图主体 | 用户当前位置、正常充电场站点位、价格 / 距离小卡片 |
| 地图工具 | 定位 icon、去充电按钮 |
| 底部弹层 | 点击场站后展示场站卡片 |

司机进入地图页后，系统获取用户手机定位。定位成功时，系统根据用户当前位置计算场站距离，并在地图上展示场站价格 / 距离小卡片。

定位失败时，地图仍展示正常场站点位，但不展示价格 / 距离小卡片中的距离信息。

地图页不展示距离排名数字。距离优先级排序只在场站列表页生效。

| 工具 | 功能说明 |

| 定位 icon | 点击后回到用户当前位置，并刷新地图数据 |

| 加载失败 | 停留在地图页，提示“场站列表加载失败，请稍后重试” |

### 6.3 场站价格 / 距离小卡片

定位成功时，地图上正常场站展示价格 / 距离小卡片。小卡片仅展示当前时段电价和距离，不展示场站名称。司机点击价格 / 距离小卡片或场站图标后，底部弹出场站信息卡片。

| 场景 | 展示示例 |
| --- | --- |
| 地图定位成功 | ¥0.8/度 / 8.6km |
| 用户定位失败 | 不展示价格 / 距离小卡片，仅展示场站图标 |

### 6.4 地图底部场站卡片

司机点击场站图标或价格 / 距离小卡片后，地图底部弹出场站卡片。点击地图空白区域或卡片关闭按钮可收起卡片。

| 展示顺序 | 小程序字段 | 后台来源 | 示例 |
| --- | --- | --- | --- |
| 1 | 场站名称 | 区域名称 / 名称 | 来充电重卡充电站-勐养站 |
| 2 | 距离 | 用户定位 + 经纬度计算 | 距你 8.6km / 距你约 8.6km |
| 3 | 当前时段电价 | 电价 / 充电价格 | 当前 ¥0.8/度 |
| 4 | 地址 | 位置 | 云南省西双版纳傣族自治州景洪市大渡岗乡213国道野象谷 |
| 5 | 充电桩数量 | 充电桩数量 | 充电桩：10 台 |
| 6 | 操作按钮 | 前端操作 | 导航、复制地址 |

### 6.5 场站列表页

场站列表页为独立页面，由地图页「去充电」按钮进入。页面支持关键词搜索、省份筛选、城市筛选、距离筛选、距离排序和导航快捷操作。

| 筛选 | 规则 | 数据来源 |
| --- | --- | --- |
| 关键词 | 匹配场站名称、地址 | 区域名称 / 名称、位置 |
| 省份 | 来源于后台正常场站所属省份 | 所属省份 |
| 城市 | 根据所选省份联动 | 所属城市 |
| 距离 | 全部距离、10km、20km、50km；默认全部距离 | 用户定位 + 经纬度计算 |
| 多条件关系 | 多个筛选条件同时生效，取交集 | 系统规则 |

#### 6.5.1 场站列表排序规则

| 场景 | 排序规则 |
| --- | --- |
| 用户定位成功 | 按用户当前位置到场站的距离从近到远排序 |
| 用户定位失败 | 按省份、城市、场站名称排序 |
| 筛选后 | 筛选结果仍按当前排序规则展示 |
| 百度驾车距离失败 | 单场站降级为直线距离，并参与排序 |

#### 6.5.2 列表行展示

| 展示内容 | 后台来源 / 计算来源 |
| --- | --- |
| 场站名称 | 区域名称 / 名称 |
| 距离 | 用户定位 + 经纬度计算 |
| 当前时段电价 | 电价 / 充电价格 |
| 地址 | 位置 |
| 充电桩数量 | 充电桩数量 |
| 导航按钮 | 经纬度、位置 |

### 6.6 外部导航

导航入口包括地图底部场站卡片、场站列表页。点击导航前系统需校验场站状态。场站状态正常时，弹出地图软件选择。

1. 高德地图；

2. 百度地图；

3. 腾讯地图。

每次点击导航均弹出选择，导航目的地为当前充电场站，小程序仅传递目的地信息。外部地图拉起失败时，支持重新选择地图或复制地址。

## 七、业务流程

### 7.1 地图页初始化流程

1. 司机进入底部「地图」入口。

2. 系统获取用户手机定位。

3. 判断用户定位是否成功。

4. 用户定位成功时，使用用户当前位置。

5. 系统获取全部正常充电场站。

6. 系统根据当前时间匹配场站当前时段电价。

7. 系统计算用户当前位置到场站的距离。

8. 前端渲染地图点位、价格 / 距离小卡片。

9. 若用户定位失败，则进入无定位模式，仅展示场站点位。

### 7.2 定位 icon 点击流程

1. 司机点击地图页定位 icon。

2. 定位 icon 进入 loading 状态。

3. 系统重新获取用户手机定位。

4. 判断用户定位是否成功。

5. 用户定位成功时，地图回到用户当前位置。

6. 系统刷新正常充电场站数据。

7. 系统重新匹配当前时段电价。

8. 系统重新计算用户当前位置到场站距离。

9. 前端更新地图点位、价格 / 距离小卡片。

10. 刷新完成后，定位 icon 恢复可点击状态。

11. 用户定位失败时，地图不移动视野，并提示“定位失败，请检查定位权限”。

### 7.3 去充电按钮点击流程

1. 司机点击地图页「去充电」按钮。

2. 「去充电」按钮进入 loading 状态。

3. 系统判断当前是否已有用户定位。

4. 若已有用户定位，则使用当前用户定位。

5. 若无用户定位，则尝试重新获取用户手机定位。

6. 系统加载全部正常充电场站。

7. 系统根据当前时间匹配场站当前时段电价。

8. 用户定位成功时，系统计算用户当前位置到场站距离。

9. 系统按距离从近到远排列场站列表。

10. 用户定位失败时，系统按省份、城市、场站名称排序。

11. 场站列表加载成功后，进入场站列表页。

12. 场站列表加载失败时，停留地图页，并提示“场站列表加载失败，请稍后重试”。

13. 加载完成后，「去充电」按钮恢复可点击状态。

### 7.4 地图点位查看流程

1. 司机点击场站图标或价格 / 距离小卡片。

2. 地图底部弹出场站卡片。

3. 卡片展示名称、距离、电价、地址、充电桩数量。

4. 司机可点击导航、复制地址。

5. 点击地图空白处或关闭按钮，收起底部卡片。

### 7.5 场站列表流程

1. 司机在地图页点击「去充电」按钮。

2. 系统加载场站列表。

3. 进入独立场站列表页。

4. 默认展示全部正常场站。

5. 定位成功时，按距离从近到远排序。

6. 定位失败时，按省份、城市、场站名称排序。

7. 司机可通过关键词、省份、城市、距离筛选场站。

8. 点击导航按钮拉起地图选择弹窗。

9. 点击返回按钮返回地图页。

### 7.6 导航流程

1. 司机点击导航。

2. 系统校验场站状态。

3. 场站正常时，弹出地图选择。

4. 司机选择高德、百度或腾讯。

5. 系统拉起外部地图。

6. 外部地图目的地为当前充电场站。

7. 拉起失败时，提示重新选择或复制地址。

## 十、处理规则

### 10.1 前后端字段一致性规则

1. 小程序展示字段必须来源于后台现有站点管理字段或系统计算字段。

2. 后台未维护的字段，小程序第一期不得展示。

3. 系统计算字段仅限距离、距离文本、距离类型、当前时段电价、定位状态。

4. 当前时段电价必须基于后台「充电价格」分时段配置计算。

5. 距离必须基于用户手机定位和后台场站经纬度计算。

6. 导航必须基于后台经纬度和位置字段。

7. 若后台字段为空，小程序按空值规则处理，不得自行生成虚假数据。

### 10.2 场站展示规则

1. 场站类型为充电场站，或满足站点管理中充电场站数据规则；

2. 场站状态为正常；

3. 经纬度完整；

4. 场站基础信息完整；

5. 充电价格配置合法。

6. 满足条件的场站展示在地图页、列表页，并参与场站列表排序。

### 10.3 场站状态规则

| 状态 | 地图页 | 列表页 | 场站列表排序 |
| --- | --- | --- | --- |
| 正常 | 展示 | 展示 | 参与 |
| 停用 | 隐藏 | 隐藏 | 不参与 |

后台将场站停用后，司机端重新进入或刷新后同步隐藏。司机点击导航前，系统二次校验场站状态。

### 10.4 经纬度规则

1. 后台新增、编辑、导入场站时，经度和纬度均必填，格式需合法。经纬度用于司机端地图展示、距离计算、外部导航目的地传递。

### 10.5 分时电价规则

1. 价格必填；

2. 支持分时段价格；

3. 分时段必须覆盖 00:00-24:00；

4. 同一场站价格时段不可重叠；

5. 同一场站价格时段不可存在缺口；

6. 价格最多支持 2 位小数；

7. 价格需为大于等于 0 的数字。

### 10.6 定位规则

1. 本期定位来源仅使用用户手机定位。

2. 进入地图页时，系统尝试获取用户手机定位。

3. 点击定位 icon 时，系统重新获取用户手机定位。

4. 点击「去充电」按钮时，优先使用当前已有用户定位；无定位时尝试重新获取用户定位。

5. 用户定位成功时，按用户当前位置计算距离。

6. 用户定位失败时，地图页仅展示场站点位。

7. 用户定位失败时，场站列表按省份、城市、场站名称排序。

### 10.7 距离计算规则

| 规则 | 说明 |
| --- | --- |
| 定位来源 | 用户手机定位 |
| 地图页作用 | 展示距离文本 |
| 列表页作用 | 按距离从近到远排序 |

| 距离类型 | 展示规则 |
| --- | --- |
| 驾车距离 | 8.6km |
| 用户定位失败 | 定位后可查看 |

### 10.8 场站列表排序规则

1. 场站列表按用户当前位置到场站的距离从近到远排序。

2. 排序只作用于场站列表页。

3. 地图页仅展示场站点位和价格 / 距离小卡片。

4. 用户定位失败时，场站列表按省份、城市、场站名称排序。

5. 筛选条件生效后，筛选结果仍按当前排序规则展示。

### 10.9 地图工具规则

1. 地图页展示两个工具：定位 icon、去充电按钮。

2. 定位 icon 用于回到用户当前位置并刷新地图数据。

3. 去充电按钮用于加载场站列表并进入场站列表页。

4. 定位 icon 和去充电按钮点击后均需进入 loading 状态。

5. loading 过程中不可重复点击。

6. 定位失败时，地图不强制移动视野。

7. 场站列表加载失败时，不进入列表页。

8. 定位失败、刷新失败、场站列表加载失败均需给出轻提示。

### 10.10 去充电按钮规则

1. 点击「去充电」按钮后，系统加载场站列表。

2. 若当前已有用户定位，则直接使用当前定位计算距离和排序。

3. 若当前无用户定位，则尝试重新获取用户手机定位。

4. 定位成功时，场站列表按距离从近到远排序。

5. 定位失败时，场站列表按省份、城市、场站名称排序。

6. 场站列表加载成功后进入场站列表页。

7. 场站列表为空时，进入列表页并展示空状态。

8. 场站列表加载失败时，停留地图页并提示失败原因。

### 10.11 刷新规则

| 刷新内容 | 说明 |
| --- | --- |
| 用户定位 | 重新获取用户手机定位 |
| 场站基础数据 | 正常充电场站列表 |
| 当前时段电价 | 根据当前时间重新匹配 |
| 距离 | 根据用户当前位置重新计算 |

### 10.12 空值处理规则

| 字段 | 空值处理 |
| --- | --- |
| 场站名称 | 不允许为空，后台保存时校验 |
| 经度 / 纬度 | 不允许为空，空值场站不进入小程序展示 |
| 位置 | 不允许为空，后台保存时校验 |
| 充电价格 | 不允许为空，后台保存时校验 |
| 充电桩数量 | 不允许为空，后台保存时校验 |
| 标准停留时长 | 不允许为空，后台保存时校验 |
| 所属省份 | 不允许为空，后台保存时校验 |
| 所属城市 | 允许为空时，小程序筛选中不进入城市筛选项 |
| 距离 | 用户定位失败时展示“定位后可查看” |
| 当前时段电价 | 匹配失败时展示“暂无电价”或不展示价格，建议后台禁止保存不完整价格 |

## 十一、功能逻辑

### 11.1 地图页核心逻辑

进入地图页 → 获取用户手机定位 → 判断用户定位是否成功 → 用户定位成功则使用用户当前位置 → 获取正常场站 → 匹配当前时段电价 → 计算场站距离 → 渲染地图点位、价格 / 距离小卡片

### 11.2 定位 icon 逻辑

点击定位 icon → 按钮进入 loading → 重新获取用户手机定位 → 用户定位成功则地图回到用户当前位置 → 刷新场站数据 → 刷新当前时段电价 → 重新计算距离 → 更新地图展示 → 按钮恢复可点击

### 11.3 去充电按钮逻辑

点击去充电 → 按钮进入 loading → 判断是否已有用户定位 → 有定位则使用当前定位 → 无定位则尝试重新获取用户定位 → 加载正常场站列表 → 匹配当前时段电价 → 定位成功则计算距离并按距离从近到远排序 → 定位失败则按省份、城市、场站名称排序 → 加载成功进入场站列表页 → 按钮恢复可点击

### 11.4 场站列表排序逻辑

可展示场站 = 正常状态 + 经纬度完整；当前位置 = 用户手机定位；距离 = 百度驾车距离 / 直线距离；排序 = 按最终可用距离从近到远；排序作用范围 = 场站列表页

### 11.5 导航逻辑

点击导航 → 校验场站状态 → 状态正常则弹出地图选择 → 司机选择地图 App → 拉起外部地图 → 目的地为当前场站 → 拉起失败则重新选择或复制地址

## 十二、异常说明

| 异常场景 | 处理 |
| --- | --- |
| 用户定位成功 | 地图页展示距离小卡片；场站列表按距离从近到远排序 |
| 用户定位失败 | 地图页仅展示场站图标；场站列表按省份、城市、场站名称排序 |
| 点击定位 icon 定位失败 | Toast 提示“定位失败，请检查定位权限” |
| 点击定位 icon 刷新失败 | Toast 提示“刷新失败，请稍后重试” |
| 点击去充电时无用户定位 | 尝试重新获取用户定位 |
| 点击去充电后定位失败 | 进入场站列表页，列表按省份、城市、场站名称排序 |
| 点击去充电后列表加载失败 | 停留地图页，Toast 提示“场站列表加载失败，请稍后重试” |
| 场站列表为空 | 进入列表页，展示“暂无可用充电场站” |
| loading 过程中重复点击 | 不重复触发，按钮保持 loading 或禁用态 |
| 无定位时地图页 | 只展示场站图标 |
| 无定位时底部卡片 | 展示价格，距离展示“定位后可查看” |
| 无定位时列表页 | 按省份 + 城市 + 场站名称排序 |
| 无定位时距离筛选 | 置灰不可用 |
| 百度距离计算失败 | 单场站降级为直线距离，展示“约 X km” |
| 场站被停用 | 刷新后隐藏，导航前拦截 |
| 导航 App 未安装 | 提示重新选择地图或复制地址 |
| 分时电价未覆盖全天 | 后台保存 / 导入失败 |
| 分时电价时段重叠 | 后台保存 / 导入失败 |
| 价格为空 | 后台保存 / 导入失败 |
| 经纬度为空 | 后台保存 / 导入失败 |
| 后台字段为空但非必填 | 小程序按空值规则隐藏对应模块或展示默认文案 |
| 后台无字段来源 | 小程序不得展示该字段 |
